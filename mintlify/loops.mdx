QUALITY.md is built around a set of nested loops that move quality judgment up
the loop stack instead of relying on ad hoc prompts and reactive reviews.

## The loops

{/* TODO: Describe each loop and its cadence. Keep the named-loop capitalization
    (Inner Loop, Middle Loop, Meta Loop, Outer Loop). */}

- **The Middle / Meta Loop** (hourly / daily) — evaluate, then review the report.
- **The Inner Loop** (continuously) — act on handoff-ready recommendations.
- **The Outer Loop** (daily / weekly) — improve the model and compound learning.

## The workflow

{/* TODO: Expand the four steps. */}

<Steps>
  <Step title="Evaluate">
    Run `/quality evaluate` to produce an evaluation report.
  </Step>
  <Step title="Review">
    Read ratings, findings, and recommendations by area or factor.
  </Step>
  <Step title="Act">
    Implement or delegate handoff-ready recommendations.
  </Step>
  <Step title="Improve">
    Run `/quality improve` to refine `QUALITY.md`.
  </Step>
</Steps>
