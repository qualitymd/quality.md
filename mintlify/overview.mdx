QUALITY.md is an open format, agent skill, and CLI for modeling a project's
quality so teams and agents can evaluate it, stay aligned on what good means,
and improve it over time.

## The three surfaces

{/* TODO: Describe the format, the /quality skill, and the qualitymd CLI, and
    that the experience is skill-first — users mostly work through the agent. */}

- **The format** — a `QUALITY.md` file that declares a quality model.
- **The /quality skill** — the primary way you work with QUALITY.md, through an
  AI assistant or coding agent.
- **The qualitymd CLI** — the deterministic tooling the skill builds on.

## The QUALITY.md file

{/* TODO: Explain the two layers and when each matters. */}

A QUALITY.md file has two layers:

1. **YAML frontmatter** — the structured quality model.
2. **Markdown body** — the context, scope, and rationale that help people and
   agents interpret and evaluate the model.
