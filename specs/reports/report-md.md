# report.md

`report.md` is the complete human Evaluation Report generated by
`qualitymd evaluation report build`.

Shared report-model, trust-boundary, active/superseded, and typed-state
invariants live in
[Evaluation report outputs](../evaluation-records/report-outputs.md).
The concise artifact is specified in [report-summary.md](report-summary-md.md), and
the machine-readable artifact is specified in [report.json](report-json.md).

This document uses BCP 14 keywords only for testable conformance requirements.
The key words "MUST", "MUST NOT", "SHOULD", and "MAY" are to be interpreted as
described in [RFC 2119](../../docs/reference/rfc2119.md) and
[RFC 8174](../../docs/reference/rfc8174.md) when, and only when, they appear in
all capitals.

## Background / Motivation

The full report is the human audit trail. It should still be summary-first for
reviewers, but unlike `report-summary.md`, it must preserve the detailed Area,
Factor, Requirement, Finding, rationale, recommendation, and record-reference
trail needed to verify or continue the evaluation.

## Scope

Covered: complete human Markdown report shape, summary-first ordering, detailed
Area/Requirement/Finding/Advice sections, Area breakdown reuse, and
reader-facing labels.

Deferred / non-goals: machine-readable JSON shape, report gate behavior, rating
roll-up semantics, and evaluation-record write contracts.

## Requirements

`report.md` **MUST** be derived from the same assembled report model as
`report-summary.md` and `report.json`. It **MUST NOT** reread evaluated source,
recompute ratings, invent findings, infer roll-ups, or choose new
recommendations.

`report.md` **MUST** present a summary-first human report before detailed audit
sections.

`report.md` **MUST** include the in-scope root verdict, scope, selected findings
and limitations, evidence basis, next action, and Area breakdown before detailed
Area and Requirement sections.

The Area breakdown **MUST** use the same compact Area breakdown data or rendering
rules as `report-summary.md`, including the Area Breakdown `Path` display value.

The full report **MUST** preserve detailed Area entries with Area-only rating,
Area-with-descendants rating, Factor rating results, rationales, analysis record
references, and not-assessed requirements when present.

Detailed Area entries **MUST** render `Path` as the Area display value: `/` for
the root Area and `<area-path>` for descendants.

The full report **MUST** preserve per-Requirement summaries with Area,
rating result, rationale, active/superseded state, and assessment-result record
references.

The full report **MUST** preserve selected and full Finding summaries with
assessment-result record references and locators when recorded.

The full report **MUST** preserve Advice/Recommendation entries, including
superseded recommendations as audit/detail data while keeping primary next
actions active-only.

Human Markdown **MUST** resolve Model, Area, Factor, and Rating Level titles from
the run's `model.md` snapshot, falling back to stable identifiers only when a
title is unavailable or traceability requires it.

`not assessed`, active, superseded, and area-group states **MUST** remain
visibly distinct in human Markdown.
