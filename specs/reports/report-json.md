# report.json

`report.json` is the machine-readable Evaluation Report generated by
`qualitymd evaluation report build`.

Shared report-model, trust-boundary, active/superseded, and typed-state
invariants live in
[Evaluation report outputs](../evaluation-records/report-outputs.md).
The human artifacts are specified in [report-summary.md](report-summary-md.md) and
[report.md](report-md.md).

This document uses BCP 14 keywords only for testable conformance requirements.
The key words "MUST", "MUST NOT", "SHOULD", and "MAY" are to be interpreted as
described in [RFC 2119](../../docs/reference/rfc2119.md) and
[RFC 8174](../../docs/reference/rfc8174.md) when, and only when, they appear in
all capitals.

## Background / Motivation

`report.json` is the stable agent/tool surface for a rendered evaluation report.
It should expose the same result as the Markdown artifacts without requiring
consumers to scrape prose, infer typed states from nulls, or join detail arrays
to recover the compact summary layer.

## Scope

Covered: generated JSON report shape, stable identifiers, typed states,
non-null collections, compact Area summary data, selected finding summaries,
recommendation references, and next-action state.

Deferred / non-goals: full assessment finding detail, raw evidence expansion,
Markdown labels, gate exit semantics, and rating roll-up semantics.

## Requirements

`report.json` **MUST** be derived from the same assembled report model as
`report-summary.md` and `report.md`. It **MUST NOT** reread evaluated source,
recompute ratings, invent findings, infer roll-ups, or choose new
recommendations.

`report.json` **MUST** carry `schemaVersion`.

`report.json` **MUST** preserve stable machine identifiers from evaluation
records, including ordered `areaPath` entries, ordered `factorPath` entries,
rating `level` ids, and record references. It **MUST NOT** replace those stable
identifiers with human display titles.

`report.json` **MAY** add derived qualified model-reference strings, but it
**MUST NOT** persist unqualified references. It **MUST** preserve `areaPath` and
`factorPath` arrays as the durable source of truth.

Collections **MUST** render as arrays, including empty arrays; they **MUST NOT**
render as `null`.

The report **MUST** expose the in-scope root verdict as a typed rating result.

The report **MUST** expose non-null scope data with recorded scope text,
narrowing, in-scope Areas, out-of-scope Areas, and missing metadata entries when
available.

The report **MUST** expose a compact `areaSummary` layer. Each Area summary
**MUST** include stable `areaPath`, typed Area-only rating state,
Area-with-descendants rating result, covered requirement count, Factor rating
results, and a note when relevant.

Area-only rating states **MUST** distinguish rated, not-assessed, and structural
area-group states. Rated and not-assessed states include the underlying rating
result; structural area-group states do not.

Factor rating results in `areaSummary` and detailed Area entries **MUST** carry
stable `factorPath` values and typed rating results.

Assessment result summaries **MUST** indicate whether each assessment result is
active or superseded through a typed lifecycle `state`.

Recommendation summaries **MUST** indicate whether each recommendation is active
or superseded through a typed lifecycle `state`. The report next action
**MUST** choose from active recommendations only.

Finding summaries **MUST** preserve canonical severity `level` and corresponding
display `title`, plus assessment-result record references and locators when
recorded.

Missing report metadata **MUST** render as typed objects with a stable `field`
and human `title`, not as prose-only strings.
