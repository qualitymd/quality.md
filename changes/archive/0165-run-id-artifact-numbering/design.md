# Run IDs and Artifact Numbering — design doc

Design for [Run IDs and Artifact Numbering](../0165-run-id-artifact-numbering.md)
and its [functional spec](spec.md).

## Context

The work splits cleanly into three movements of very different weight:

- **One new mechanism** — the run `id`. This is the only thing that adds code.
- **One structural rename** — recommendation `id` (`QREC`) → `number`, whose only
  real work is retargeting the data path, query selector, and cross-references.
- **Net deletion** — the ranked-finding `QFIND` ID and the stability machinery
  that existed only to keep it stable.

Today identity flows through three helpers in `internal/evaluation/data.go`:
`evaluationArtifactID`, `runArtifactID`, and `parseArtifactID`, which encode the
`QEVAL-NNNN` / `QREC-NNNN-NNN` / `QFIND-NNNN-NNN` grammar. `normalizeArtifactIDs`
assigns recommendation and finding IDs in `SetData` before path derivation and
effective validation. Reports render the tokens in `report_tree.go`
(`reportRunLine`, the typed-reference builder, the Top Findings and recommendation
tables). The run number itself is allocated by `nextRunNumber` (`path.go`) as
`max + 1` over the directory — unchanged by this case.

## Approach

### Run ID generation

Generate the run ID in `create.go` where the `RunManifest` is built, alongside
`newRunCreatedAt()`. Take a single `time.Now().UTC()` and format both fields from
it: `createdAt` as RFC 3339 and the ID timestamp as ISO-8601 basic
(`YYYYMMDDThhmmssZ`). The nanoid tail is a 12-character lowercase base32 string
(an ambiguity-free Crockford-style alphabet, no `i`/`l`/`o`/`u`) read from
`crypto/rand`. The full ID is `<timestamp>-<nanoid>`, e.g.
`20260629t103745z-k7m2p9qx4n3a`.

No directory scan and no retry: unlike the run number, the ID needs no
coordination — entropy carries uniqueness. Add `id` to the `RunManifest` struct
(`types.go`) and write it through unchanged. Readers treat it as opaque; the only
invariant is non-empty.

Generation pulls a tiny seam: a package-level `now func() time.Time` is not
required, but the timestamp+random pair must be injectable enough for fixtures —
see Fixtures below.

### Recommendation numbering

`assignRecommendationArtifactIDs` keeps its shape (sequential next-unused,
preserve-supplied-valid) but operates on a `number` integer field instead of a
`QREC` string. `existingRecommendationArtifactIDs` walks the same directory and
reads `number`. `recommendationDataPath` takes the number and zero-pads it to
`data/advice/recommendations/<NNN>/recommendation-result.json`;
`dataPathForQuery`'s recommendation `--selector` becomes the number.
`validateRankedRecommendations` / `validateFindingCoverage` key their
`recommendations` map by number, and payload `recommendationRef` /
`recommendationRefs` values carry the number.

The `QEVAL`/`QREC`/`QFIND` parse/format helpers collapse to a zero-pad formatter
and a positive-integer check.

### Finding artifact-ID removal

Delete `assignFindingArtifactIDs`, `assignFindingRankingArtifactIDs`,
`existingFindingArtifactIDs`, the QFIND effective validation, and the contract
field. `normalizeArtifactIDs` drops its finding call and touches recommendations
only. `artifactFindingRefKey` becomes dead and is removed. `FindingRankingResult`
keeps `orderedFindings` with `rank`/`findingRef`/`tier`/`rationale`.

### Rendering

`reportRunLine` renders `Run <NNNN>` from the number and adds a copyable
`Run ID:` field carrying `RunManifest.id`. The typed-reference builder emits
`evaluation:<run-id>/recommendation/<number>`. The recommendation tables render a
`#` column from the number; the Top Findings table drops its ID column (the
finding cell stays a link to the payload-local `#finding-<id>` anchor).

### Schema

`data_contract.go` is the typed source of truth; edit it (run `id`, recommendation
`number` integer, removed finding `id`) and regenerate
`evaluation-data.schema.json`.

### Fixtures

The report gallery is regenerated by `scripts/report-gallery`. Because the run ID
is random, the gallery must pin it deterministically — extend `pinRunCreatedAt`
(which already post-pins `createdAt`) to also pin a fixed run ID, so report output
is byte-stable. Unit tests that assert on rendered reports use the same pinned ID.

## Spec response

- Run ID generation is centralized at run creation, shares the `createdAt`
  instant, and is independent of the number (requirements 1, 2).
- The ID is opaque on read; no grammar is parsed (requirement 3).
- The number remains the friendly directory/header identity, with the ID surfaced
  as a copyable field (requirements 4, 11).
- Recommendation numbering stays mechanical and self-contained, with paths and
  references derived from the number (requirements 5–8, 12).
- Finding identity reduces to its requirement-scoped selector; anchors are
  untouched (requirements 9–11).
- The old grammar and any migration path are removed; the gallery is regenerated
  (requirement 13).

## Alternatives

- **UUIDv7 for the run ID.** Rejected: it embeds a timestamp and sorts, but the
  hex/dash form is less readable and transcribable than a date plus a short
  reduced-alphabet tail, and it would add a dependency or its own hand-roll for no
  ergonomic gain. The chosen form gives the same sortability with a friendlier
  token.
- **Replace the run number with the ID as the directory name.** Rejected: it
  wrecks directory browsing and in-repo "run 3" reference. The number and ID solve
  different jobs; keep both.
- **`math/rand` for the tail.** Rejected: seed-predictable, reintroducing exactly
  the low-improbability risk this case removes. Use `crypto/rand`.
- **Keep a globally-unique recommendation ID too.** Rejected: unnecessary — a
  recommendation is addressed through the run ID, so a short per-run number
  suffices and reads better.
- **Keep the `QFIND` finding number.** Rejected: it is a second identity for a
  thing that already has a good one, encodes only an unstable rank, and forces the
  stability-by-`findingRef` machinery this case deletes.
- **Tolerate pre-change manifests without an ID (fallback rendering).** Rejected:
  early-alpha clean break with no migration; the gallery is regenerated and
  pre-change runs are out of scope.

## Trade-offs & risks

- **Non-deterministic creation.** `create` now has two non-deterministic sources
  (time and randomness) instead of one. Mitigated by the existing pinning seam:
  fixtures and tests pin both `createdAt` and the run ID.
- **Bare-number references.** `recommendationRef` carries a plain number rather
  than a self-describing `QREC` token. Acceptable because it is a typed field
  resolved within one run; the typed report reference still spells out
  `evaluation:<run-id>/recommendation/<number>`.
- **Lost terse multi-finding handoff.** "Fix finding 3" is no longer expressible
  as a global token. Acceptable because findings are handed off through
  recommendations (which keep numbers); a non-advice-driving finding needs no
  handoff token at all.
